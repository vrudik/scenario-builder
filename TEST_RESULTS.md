## Созданные тесты

### Unit тесты
1. **scenario-spec.test.ts** - тесты валидации спецификаций
   - Валидация корректной спецификации
   - Отклонение некорректной спецификации
   - Парсинг валидной спецификации

2. **scenario-builder.test.ts** - тесты компилятора
   - Компиляция spec в workflow graph
   - Генерация политик исполнения
   - Генерация deployment descriptors
   - Canary для высокорисковых сценариев

3. **tool-registry.test.ts** - тесты реестра инструментов
   - Регистрация инструментов
   - Получение инструментов
   - Фильтрация по риск-классу
   - Удаление инструментов
   - Поиск инструментов

4. **tool-gateway.test.ts** - тесты шлюза инструментов
   - Выполнение в sandbox режиме
   - Отклонение запрещенных инструментов
   - Rate limiting

### Ручной тест
**test-manual.ts** - комплексный тест всех компонентов
- Проверка всех основных функций без необходимости установки зависимостей

## Как запустить тесты

### Вариант 1: С установкой зависимостей (рекомендуется)

```bash
# Установка зависимостей
npm install

# Запуск всех тестов
npm test

# Запуск с покрытием
npm test -- --coverage

# Запуск в watch режиме
npm test -- --watch

# UI интерфейс для тестов
npm test -- --ui
```

### Вариант 2: Без установки зависимостей

```bash
# Ручной тест (использует npx для временной установки tsx)
npx tsx test-manual.ts

# Или из корня проекта с указанием пути
cd "C:\Всякое\Конструктор сценариев"
npx tsx test-manual.ts
```

### Вариант 3: Проверка типов

```bash
# Проверка без компиляции
npm run typecheck

# Или напрямую
npx tsc --noEmit
```

## Известные проблемы

1. **Проблема с путями в PowerShell**: 
   - При использовании PowerShell могут возникать проблемы с кириллическими путями
   - Решение: использовать полные пути или перейти в директорию проекта перед выполнением команд

2. **Зависимости не установлены**:
   - Для полного тестирования необходимо выполнить `npm install`
   - Ручной тест (`test-manual.ts`) может работать без установки через `npx tsx`
