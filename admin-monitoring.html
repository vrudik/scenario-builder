🚦 Canary, deployment lane и OPA

Полоса трафика (stable / canary) выбирается из deployment descriptor и попадает в контекст инструментов, события и (при необходимости) вход OPA. Shadow-стратегия дублирует часть запусков на canary без влияния на основной ответ.

  • Настройка allow/block списков по полосе: Конструктор — поля canaryAllowedTools, canaryBlockedToolIds, stableBlockedToolIds.
  • Наблюдение за run и deploymentLane в данных: Выполнения (tenant — в шапке админки; опционально live по WebSocket /ws/admin-runs).
  • Выкат образа и секреты (не только GHCR): в репозитории docs/guides/DEPLOY_CI_GITHUB.md, workflow Deploy staging.
  • Подробнее про Temporal vs in-memory: файл docs/guides/TEMPORAL_VS_IN_MEMORY.md в репозитории.
