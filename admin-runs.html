🔁 Ход выполнения (live)

Узлы пишутся в NodeExecution при запуске с БД. События — из ExecutionEvent: оркестратор сохраняет trigger, node_completed, node_failed, compensation при наличии репозитория БД. Обновляется вместе с «Единый статус» (авто 2 с / 5 с) и при включённом «доп. опрос 1,5 с» — чаще только панели live без повторного тяжёлого unified-status. При включённом WebSocket сервер шлёт снимки по каналу /ws/admin-runs (интервалы как у авто / 1,5 с при fast).
