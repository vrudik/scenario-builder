# Быстрое решение проблемы

## Проблема: "Failed to fetch" или сервер не запускается

### Решение 1: Запуск через командную строку (cmd)

1. Откройте **командную строку** (cmd, не PowerShell)
2. Выполните:
```cmd
cd /d "C:\Всякое\Конструктор сценариев"
node server.js
```

3. Должно появиться:
```
========================================
🚀 Веб-сервер запущен!
========================================
```

4. Откройте в браузере: http://localhost:3000/test-agent.html

### Решение 2: Дважды кликните на bat файл

1. Найдите файл `start-test-server.bat` в проводнике
2. Дважды кликните на него
3. Должно открыться окно с сервером
4. Откройте в браузере: http://localhost:3000/test-agent.html

### Решение 3: Если порт 3000 занят

Измените порт в `server.js` (строка 6):
```javascript
const PORT = 3001; // или другой свободный порт
```

Затем откройте: http://localhost:3001/test-agent.html

### Решение 4: Проверка Node.js

Убедитесь, что Node.js установлен:
```cmd
node --version
```

Должно показать версию (например: v24.13.0)

### Решение 5: Открытие страницы напрямую

Если сервер не запускается, можно открыть страницу напрямую:
1. Найдите файл `test-agent.html` в проводнике
2. Дважды кликните на него
3. Страница откроется, но API не будет работать

## Проверка работы

После запуска сервера проверьте:
1. Откройте: http://localhost:3000/api/agent/status
2. Должен вернуться JSON с информацией о статусе

## Если ничего не помогает

1. Проверьте, что вы в правильной директории
2. Проверьте, что файл `server.js` существует
3. Попробуйте запустить: `node --version`
4. Попробуйте запустить: `node server.js` напрямую
