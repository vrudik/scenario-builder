## Value Props (3 pillars)

### 1. Governance by Design

Every agent action passes through policy gates. OPA policies enforce what tools can be called, cost budgets prevent runaway spend, and PII masking protects sensitive data. No bolt-on compliance — it's built in.

### 2. Spec-as-Code

Define scenarios declaratively: triggers, tools, risk classes, SLA, cost limits. Version in Git, test in CI, review in PRs. Your AI workflows get the same rigor as your application code.

### 3. Durable Execution

Temporal-backed orchestration ensures scenarios complete even after failures. Saga pattern handles compensation. In-memory runtime for development, Temporal for production — same spec, zero code changes.

## How It Works (4 steps)

1. **Define** your scenario spec — what tools, what limits, what risk class
2. **Deploy** with one command — Docker, Kubernetes, or single binary
3. **Execute** autonomously — LLM agents handle the workflow with guardrails
4. **Audit** everything — full trace of every decision, every tool call, every cost
