# Инструкция по запуску теста

## Проблема с путями

Из-за кириллицы в пути проекта (`C:\Всякое\Конструктор сценариев`) возникают проблемы с выполнением команд.

## Решение: Запуск вручную

### Вариант 1: Через командную строку (cmd)

1. Откройте командную строку (cmd)
2. Выполните:
```cmd
cd /d "C:\Всякое\Конструктор сценариев"
npm run example:ollama
```

### Вариант 2: Через PowerShell

1. Откройте PowerShell
2. Выполните:
```powershell
cd "C:\Всякое\Конструктор сценариев"
npm run example:ollama
```

### Вариант 3: Через bat файл

Дважды кликните на файл `test-agent.bat` в проводнике Windows.

### Вариант 4: Напрямую через tsx

```cmd
cd /d "C:\Всякое\Конструктор сценариев"
npx tsx examples/agent-ollama-example.ts
```

## Что должно произойти

1. ✅ Проверка Ollama (должен быть запущен)
2. ✅ Проверка модели llama3.2:1b
3. ✅ Запуск Agent Runtime
4. ✅ Выполнение теста с локальной моделью

## Если Ollama не запущен

Запустите Ollama вручную или через командную строку:
```cmd
"%LOCALAPPDATA%\Programs\Ollama\ollama.exe"
```

## Ожидаемый результат

Вы должны увидеть:
- Сообщения о проверке Ollama
- Запуск агента
- Результаты выполнения с количеством токенов и tool calls
